1.6 在WHERE子句中引用别名列

一般情况下这种是无法执行的

解决方案:

适用：

  • 聚合函数
  • 标量子查询
  • 窗口函数
  • 别名

将含有别名列的查询放入内嵌视图，就可以在外层查询中引用别名列。
原因： WHERE子句会比SELECT子句先执行。
其中： 本例中内嵌视图的别名 x 是因为有些数据库必须给内嵌视图取名。

1.7 串联多列的值

原表：

ENAME JOB
CLARK MANAGER
KING PRESIDENT
MILLER CLERK

目标结果：
ENAME |
—|
CLARK WORKS AS A MANAGER |
KING WORKS AS A PRESIDENT |
MILLER WORKS AS A CLERK|

解决方案：

DB2、Oracle和PostgreSQL

这些数据库用双竖线作为串联运算符。

MySQL

该数据库使用CONCAT函数。

SQL Server

该数据库使用”+”作为串联运算符。

1.8 在SELECT语句里使用条件逻辑

CASE表达式能对查询结果执行条件逻辑判断。其中，STATUS就是CASE表达式执行结果的别名。

1.9 限定返回行数

DB2

使用FETCH FIRST 子句

MySQL 和PostgreSQL

使用LIMIT 子句

Oracle

对于Oracle而言，通过在WHERE子句中限制ROWNUM的值来获得指定行数的结果集。
不能以此来返回第五行数据。

SQL Server

使用top关键字限定返回行数。

1.10 随机返回若干行记录

DB2

把内置函数RAND 和 ORDER BY 、FETCH 结合使用

MySQL

把内置函数RAND和LIMIT、ORDER BY 结合使用。

PostgreSQL

把内置函数RANDOM和LIMIT、ORDER BY 结合使用。

Oracle

在内置包 DBMS_RANDOM 里可以找到VALUE函数，把该内置函数和ORDER BY、内置函数ROWNM结合使用。

SQLServer

同时使用内置函数NEWID和TOP、ORDER BY 来返回一个随机结果集。

1.11 查找Null值

NULL 值不会等于或者不等于任何值，甚至不能与其自身作比较。因此不能用=或！= 来测试某一列的值是否为NULL。可以用IS　NOT NULL来找到给定列的值不是NULL的所有行。

1.12 把NULL值转换为实际值

使用COALESCE 函数将NULL值替代为实际值。

使用CASE语句

尽管CASE也能把NULL 值转换为实际值，但COALESCE函数更加方便简洁。
